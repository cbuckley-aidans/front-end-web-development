USER STORIES DESIGN

WHAT IS A USER STORY?

User stories help translate technical requirements into clear, user-focused descriptions of functionality. They follow a specific format:

As a [type of user], I want to [goal], so that I can [benefit].

This simple structure ensures that all development work focuses on delivering value to users rather than implementing technology for its own sake.

USER STORY ELEMENTS

1

USER ROLE

Identifies who will use the feature and their context.

  • Registered user
  • Admin user
  • Guest visitor
  • Content creator
  • Customer support
2

USER GOAL

What the user wants to accomplish.

  • Search for content
  • Update profile
  • Save preferences
  • Share information
  • View statistics
3

USER BENEFIT

Why the user wants this feature.

  • Save time
  • Reduce errors
  • Make decisions
  • Stay informed
  • Achieve goals

CREATING USER STORIES

1

IDENTIFY USERS

Who will use this feature? Be specific:

  • Student
  • Teacher
  • Admin
  • Guest
  • Parent
2

DEFINE GOALS

What do they want to do?

  • View grades
  • Submit assignments
  • Message teachers
  • Check schedule
  • Download resources
3

SPECIFY BENEFITS

Why do they want this?

  • Track progress
  • Save time
  • Avoid mistakes
  • Improve learning
  • Plan effectively

FROM USER STORY TO DESIGN CRITERIA

USER STORY

As a student, I want to search for assignments by subject so that I can find my work quickly.

DESIGN CRITERIA

  • The website must allow filtering assignments by subject
  • Search results must appear within 3 seconds
  • Results must be sorted by due date by default
  • Each result must show the assignment name, class, and due date
